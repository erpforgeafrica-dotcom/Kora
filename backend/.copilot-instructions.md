# Backend Development Instructions

## Architecture

**Pattern**: Modular monolith with Express.js + TypeScript  
**Database**: PostgreSQL with manual migrations  
**Queue**: BullMQ + Redis for async processing  
**Auth**: Clerk session token verification

## Module Structure

```
backend/src/modules/{module}/
├── routes.ts       ← Express routes
├── service.ts      ← Business logic (optional)
└── types.ts        ← TypeScript interfaces
```

## Creating a New Module

### 1. Create Route File
```typescript
// backend/src/modules/mymodule/routes.ts
import { Router } from "express";
import { queryDb } from "../../db/client.js";
import { getRequiredOrganizationId } from "../../shared/http.js";

export const myModuleRoutes = Router();

myModuleRoutes.get("/", async (req, res, next) => {
  try {
    const orgId = getRequiredOrganizationId(res, req.header("x-org-id"));
    const data = await queryDb("SELECT * FROM my_table WHERE org_id = $1", [orgId]);
    res.json({ count: data.length, data });
  } catch (err) {
    next(err);
  }
});
```

### 2. Register in app.ts
```typescript
import { myModuleRoutes } from "./modules/mymodule/routes.js";
app.use("/api/mymodule", requireAuth, myModuleRoutes);
```

## Multi-Tenancy Pattern

**CRITICAL**: Always validate organization context.

```typescript
// ✅ CORRECT
const orgId = getRequiredOrganizationId(res, req.header("x-org-id"));
const clients = await queryDb(
  "SELECT * FROM clients WHERE org_id = $1",
  [orgId]
);

// ❌ WRONG - Missing org_id filter
const clients = await queryDb("SELECT * FROM clients");
```

## Database Patterns

### Queries
```typescript
import { queryDb } from "../../db/client.js";

// SELECT
const rows = await queryDb("SELECT * FROM users WHERE id = $1", [userId]);

// INSERT with RETURNING
const result = await queryDb<{ id: string }>(
  "INSERT INTO bookings (org_id, client_id) VALUES ($1, $2) RETURNING id",
  [orgId, clientId]
);

// UPDATE
await queryDb(
  "UPDATE bookings SET status = $1 WHERE id = $2 AND org_id = $3",
  ["confirmed", bookingId, orgId]
);
```

### Migrations
```bash
# Create migration
npm run db:migrate:create my_migration_name

# Run migrations
npm run db:migrate

# Rollback
npm run db:migrate:rollback
```

## Async Processing (BullMQ)

### When to Use Queues
- ✅ Email/SMS notifications
- ✅ Report generation
- ✅ AI processing
- ✅ External API calls (slow)
- ❌ Simple CRUD operations
- ❌ Real-time responses

### Producer Pattern
```typescript
import { notificationQueue } from "../../queues/index.js";

await notificationQueue.add("send-email", {
  to: "user@example.com",
  subject: "Booking Confirmed",
  template: "booking-confirmation",
  data: { bookingId }
});
```

### Consumer Pattern
```typescript
// backend/src/workers.ts
notificationQueue.process("send-email", async (job) => {
  const { to, subject, template, data } = job.data;
  await sendEmail(to, subject, template, data);
});
```

### Role-Based Access
```typescript
// In route handler

if (userRole !== "business_admin") {
  return res.status(403).json({ error: "Forbidden" });
}
```

## Error Handling

### Standard Pattern
```typescript
myRoutes.post("/", async (req, res, next) => {
  try {
    // Your logic
    res.json({ success: true });
  } catch (err) {
    next(err); // Let errorHandler middleware handle it
  }
});
```

### Custom Errors
```typescript
if (!bookingId) {
  return res.status(400).json({ error: "Booking ID required" });
}

if (!hasPermission) {
  return res.status(403).json({ error: "Forbidden" });
}
```

## Caching Strategy

### Redis Cache
```typescript
import { redis } from "../../db/redis.js";

// Set cache (5 minutes)
await redis.set(`booking:${id}`, JSON.stringify(booking), "EX", 300);

// Get cache
const cached = await redis.get(`booking:${id}`);
if (cached) return JSON.parse(cached);
```

### When to Cache
- ✅ Frequently accessed data (dashboards)
- ✅ Expensive computations (analytics)
- ✅ External API responses
- ❌ User-specific data (privacy)
- ❌ Real-time data (bookings)

## Performance

### Pagination
```typescript
const page = parseInt(req.query.page as string) || 1;
const limit = parseInt(req.query.limit as string) || 25;
const offset = (page - 1) * limit;

const data = await queryDb(
  "SELECT * FROM bookings WHERE org_id = $1 ORDER BY created_at DESC LIMIT $2 OFFSET $3",
  [orgId, limit, offset]
);
```

### Rate Limiting
```typescript
import rateLimit from "express-rate-limit";

const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 100 // limit each IP to 100 requests per windowMs
});

app.use("/api/", limiter);
```

## Anti-Patterns

### ❌ DON'T
```typescript
// Missing org_id validation
const data = await queryDb("SELECT * FROM bookings");

// Hardcoded values
const orgId = "org_default";

// Synchronous external API calls
const result = await fetch("https://slow-api.com");
res.json(result); // Blocks response

// No error handling
const data = await queryDb("SELECT * FROM bookings");
res.json(data); // What if query fails?
```

### ✅ DO
```typescript
// Validate org_id
const orgId = getRequiredOrganizationId(res, req.header("x-org-id"));

// Use environment variables
const orgId = process.env.DEFAULT_ORG_ID;

// Queue external API calls
await apiQueue.add("fetch-data", { url });
res.json({ queued: true });

// Handle errors
try {
  const data = await queryDb("SELECT * FROM bookings WHERE org_id = $1", [orgId]);
  res.json(data);
} catch (err) {
  next(err);
}
```
